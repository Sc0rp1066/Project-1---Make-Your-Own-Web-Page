Webpage Basics
  • The Web - (World Wide Web) is collection of HTML documents.
  • HTML (Hypertext Markup Language) - The standard markup language used to create and format the content of web pages.
  • Web Page Layout- Invisible boxes with content and style formatting. Class attributes define the content of the boxes.
  • CSS - Cascading Style Sheets - Style sheet language used for describing the look and formatting of a document written in markup language.
  • DOM- Document Object Model - A cross platform and language independent convention for representing and interacting with objects in HTML. The nodes of every document are organized in a tree structure called the DOM.
  • URL - Link or Webpage address in the internet.
  • HTTP - Hyper Text Transfer Protocol - How web browsers talk to the webpage servers across the internet.
  • Web applications
HTML is the heart of the Web !

Some HTML Basics:

  • Tag - HTML tag is code that describes how a web page is formatted. Tags appear in HTML as brackets made up the less than / greater than symbols. Most have an opening tag < > and a closing tag < / >.
  • Element - HTML element is everything within and including the set of opening and closing tags.
  • Attribute - The property of and HTML element ex. <a href= "url"> content </a>.
  • Text content (what you see)
  • Markup (what the content looks like)
  • References to other documents (ex. Images, videos...)
  • Links to other pages

HTML Document Structure:

<! Doctype HTML>
<html>
<head>
<title> Title </title>
</head>
<body>
<b> Content </b>
</body>
</html>
TAG Types:

Inline Tags: Only contains text. HTML renders white space to a single space.

  • Anchor - <a href= "url"> content </a> - Create hyperlinks
  • Bold - <b> Content </b> - Bolds content
  • Emphasize - <em> content </em> - Italicize Content
  • Span - <span> content </span> - contains content with attribute
  • Strong - <strong> content </strong> - contains content. Bolds with attribute.

Block Tag: Creates "invisible boxes" around content which can have height and width elements. Used to create "container elements"

  • Paragraph - <p> content </p> - Paragraph with line break
  • Div - <div> content </div> - Used to define "boxes"
  • Form - <form> content </form>

Design and Style:

Choose the right type element for certain portions of the page. Always use the appropriate tag for the content.

Element Box Size Issues:

HTML elements are boxes and each box has 4 size components, margin, border, padding, and content dimensions.
Because of the number components in a box it can be difficult to size.

Two techniques to help with sizing.
  • Set sizes in terms of percent rather than pixels. So elements can re-size with the window size.
  • Set the "box-sizing" attribute in the CSS file to "border-box" for every element. *{box-sizing: border-box;}

Box positioning:

Divs are block elements so by default take up the entire width of the page.
Add the CSS rule "display: flex;" to the parent container will allow the divs within appear next to each other.

Routine to improve page - Code, Test & Refine:

  1. Define Structure - Boxify
    • Look at tags
    • Size of boxes
    • Look at position of boxes
  2. Look for repeated style & semantics
  3. Write HTML
  4. Apply Styles
  5. Fix things
  6. Repeat steps 3 & 4 until design is correct. Use developer tools to test changes on the fly prior to making changes to the CSS file.
